The .env (environment) file is used to store configuration settings and secrets for your application. We use it so that sensitive information (like passwords and database connection strings) isn't hardcoded directly into the source code, which would be a security risk if the code is shared or pushed to GitHub.

The .env.example file is meant to be a safe "template" that you commit to your repository so other developers know which variables they need to create in their own local .env file.

Here is exactly what to put in your actual .env file to make the password reset emails work. I'll use Gmail as the example since your config shows smtp.gmail.com:

1. FRONTEND_RESET_PASSWORD_URL
This is the URL where your frontend (React, Vue, Angular, etc.) handles the reset password screen.
What to put: If your frontend runs locally on port 3000, keep it as http://localhost:3000/reset-password. The backend will append the unique token to this URL (e.g., ?token=abc...) and send it in the email.

2. SMTP_HOST & SMTP_PORT
These point to the mail server.
What to put: Keep them as smtp.gmail.com and 587 if you want to use a Gmail account to send out your emails.

3. SMTP_USERNAME & SMTP_FROM_EMAIL
What to put: Replace your_email@gmail.com with your actual Gmail address (e.g., my.company.erp@gmail.com).

SMTP_USERNAME is used to log in.
SMTP_FROM_EMAIL is what the user sees in the "From" line when they receive the email.
4. SMTP_PASSWORD (Crucial Step for Gmail)
Google no longer allows you to use your regular Gmail password for external apps. You must generate a special App Password.
What to put: A 16-character App Password generated by Google.

How to get a Gmail App Password:

Go to your Google Account settings (Manage your Google Account).
Go to the Security tab.
Ensure 2-Step Verification is turned ON.
In the search bar at the top, search for App passwords.
Create a new App Password (you can name it "ERP Backend").
Google will give you a 16-letter password (e.g., abcd efgh ijkl mnop).
Paste that password into your .env file without the spaces: SMTP_PASSWORD=abcdefghijklmnop
5. SMTP_USE_TLS
What to put: Keep this as true. It ensures your connection to the Gmail server is secure.

Once you fill this out in your real .env file, your FastAPI server will be able to send real password reset emails whenever someone hits the /api/v1/auth/forgot-password endpoint!
